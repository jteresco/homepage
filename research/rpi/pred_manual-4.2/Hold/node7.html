General Functions for Predictive Balancing

  • void pred_set_ref_bal_func(PRED_CBPROC set_pred_ref_bal_func)

    Allows a load balancing function to be registered with pred. The load balancing function registered with the predictive library should have the following prototype format: 

    void set_pred_ref_bal_func(pMeshPB);
    Load balancing after each stage of enrichment is shut off by a call to this function.

    For example, the prototype for Iterative Tree Balancing is 

    void pmdb_load_balance(pMeshPB);
    To use ITB predictively during enrichment, you would call 
    pred_set_ref_bal_func(pmdb_load_balance);
    before the first call to refdref.

    An example of registering another load balancer may be found in Making Other Balancers Predictive.

  • void pred_unset_ref_bal_func(void)

    Used to clear the load balancing function registered with pred so that it is no longer used. Calling this function shuts off predictive balancing. Load balancing after each stage of enrichment is turned back on by a call to this function.

  • int pred_ref_balancing_on(void)

    Returns 1 if predictive balancing before refinement is turned on; 0 otherwise.

  • int pred_balancing_on(void)

    Returns 1 if predictive balancing before refinement is turned on; 0 otherwise. If extensions to allow for predictive balancing before coarsening are ever added, this will return 1 if balancing before refinement or coarsening is turned on. Currently, identical to pred_ref_balancing_on.
