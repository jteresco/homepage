Library Instances (PRED)

There are three instances of the library compiled with different flags available:

predis compiled with -g and all optimizations turned off
pred-Odoes not use the -g option and uses -O3 -qstrict
pred-timeis optimized just as pred-O is and contains code to do instrumentation; it will create a directory 'PredTimes' in your current working directory and deposit files with timings of refinement, predictive, and nonpredictive balancing. If this library is used and nonpredictive balancing is to be done during refinement, a call to refdref_set_rebal_func(pred_nonpred_balance_refdref) should be made to handle the timing of the nonpredictive balancing.

Please contact ziantzl@cs.rpi.edu if you wish to use the following instances of the library:

pred_oct contains functions to perform predictive Octree balancing; it is otherwise the same as pred
pred_oct-Ocontains functions to perform predictive Octree balancing; it is otherwise the same as pred-O
pred_oct-timecontains functions to perform predictive Octree balancing; it is otherwise the same as pred-time
