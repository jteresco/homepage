Uniform Element Workload Predictive Balancing

  • void pred_set_ref_balancing(int balType)

    Sets predictive balancing to a particular default type if called from an application code. Default types are as follows:

    PRED_NONE no predictive balancing done; standard nonpredictive ITB used during enrichment. Note that setting predictive balancing tothis type will allow timing to be performed on the nonpredictive balancing if the pred-time library is being used.
    PRED_ITB use predictive Iterative Tree Balancing. Balancing will be timed if the pred-time libarary is used.

    Using a pred_oct* instance of the library adds:

    PRED_OCT use predictive Octree Partitioning. Balancing will be timed if the pred_oct-time libarary is used.


  • void pred_check_env(void)

    Allows predictive balancing types to be set by an environment variable (PRED_REF_BAL) if called by an application code. Valid values for PRED_REF_BAL are as follows:

    0 same effect as using PRED_NONE with pred_set_ref_balancing
    1 same effect as using PRED_ITB with pred_set_ref_balancing

    Using a pred_oct* instance of the library adds:

    3 same effect as using PRED_OCT with pred_set_ref_balancing


    NOTE: In application code, any call to refdref_set_rebal_func() should be placed AFTER a call to pred_check_env() or
    pred_set_ref_balancing() as they will set the refdref balancing function to a default if the value for predictive balancing is set to
    PRED_NONE (i.e., 0).

  • int pred_ref_balancing_on(void)

    Returns 1 if predictive balancing before refinement is turned on; 0 otherwise.

  • int pred_balancing_on(void)

    Returns 1 if predictive balancing before refinement is turned on; 0 otherwise. If extensions to allow for predictive balancing before coarsening are ever added, this will return 1 if balancing before refinement or coarsening is turned on. Currently, identical to pred_ref_balancing_on.
